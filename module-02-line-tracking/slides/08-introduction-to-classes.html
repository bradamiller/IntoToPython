class LineSensor:

class — keyword that starts a class
LineSensor — name (CamelCase!)
: — colon starts the body
pass — placeholder ("nothing yet")
Parentheses () after the class name create a new object — this is called instantiation.

Step 2: The __init__ Method

__init__ runs automatically when you create an object — it's called the constructor.


class LineSensor:
    def __init__(self):
        self.reflectance = Reflectance.get_default_reflectance()
        self.threshold = 0.5
def __init__(self):
Special method — runs when object is created
self
Refers to the object being created ("me, myself")
self.threshold
Stores 0.5 on this object as an attribute 

sensor = LineSensor()       # __init__ runs here!
print(sensor.threshold)     # Prints: 0.5
self.something means "this object's something"

Step 3: Adding Methods

A method is a function inside a class. Methods can call other methods!


class LineSensor:
    def __init__(self):
        self.reflectance = Reflectance.get_default_reflectance()
        self.threshold = 0.5

    def get_left(self):
        return self.reflectance.get_left()

    def get_right(self):
        return self.reflectance.get_right()

    def get_error(self):
        return self.get_left() - self.get_right()
How sensor.get_error() works:
You
sensor.get_error() ↓ calls
self.get_left() ↓ calls
self.reflectance.get_left() ↑ returns 0.8
self.get_right() ↓ calls
self.reflectance.get_right() ↑ returns 0.3
Result
0.8 - 0.3 = 0.5 ↑ returns 0.5

Step 4: Boolean Methods

Methods that return True or False — perfect for if and while!


def is_at_cross(self):
    return self.get_left() > self.threshold and self.get_right() > self.threshold

def is_off_line(self):
    return self.get_left() < self.threshold and self.get_right() < self.threshold

Following Line

is_at_cross → False
is_off_line → False

🎯 At a Cross

⚠️ Off the Line

is_at_cross → False
is_off_line → True ✓
Sensor reading HIGH (on dark line)
Sensor reading LOW (off line)

🏆 The Complete LineSensor Class


from XRPLib.reflectance import Reflectance

class LineSensor:
    def __init__(self):
        self.reflectance = Reflectance.get_default_reflectance()
        self.threshold = 0.5

    def get_left(self):
        return self.reflectance.get_left()

    def get_right(self):
        return self.reflectance.get_right()

    def get_error(self):
        return self.get_left() - self.get_right()

    def is_at_cross(self):
        return (self.get_left() > self.threshold
                and self.get_right() > self.threshold)

    def is_off_line(self):
        return (self.get_left() < self.threshold
                and self.get_right() < self.threshold)

Before vs. After — Why Classes Win

❌ Before (Lesson 7)


reflectance = Reflectance.get_default_reflectance()
threshold = 0.5

left = reflectance.get_left()
right = reflectance.get_right()
error = left - right
at_cross = left > threshold and right > threshold
off_line = left < threshold and right < threshold
8 lines · variables scattered · hard to reuse

✅ After (Lesson 8)


sensor = LineSensor()

error = sensor.get_error()
at_cross = sensor.is_at_cross()
off_line = sensor.is_off_line()
5 lines · reads like English · reusable anywhere

🚨 Common Mistakes

Watch out for these three errors — almost everyone makes them at first!

Mistake #1: Forgetting self in the definition

❌ WRONG
def get_error():
✅ CORRECT
def get_error(self):

Mistake #2: Forgetting self. when accessing attributes

❌ WRONG
return reflectance.get_left()
✅ CORRECT
return self.reflectance.get_left()

Mistake #3: Passing self when calling a method

❌ WRONG
sensor.get_error(sensor)
✅ CORRECT
sensor.get_error()

📋 Key Takeaways

📦
Classes organize code

Group related data (attributes) and actions (methods) into one reusable package.

🏗️
__init__ sets things up

The constructor runs automatically and uses self to store data on the object.

🪞
self = "my"

Write self in definitions, never when calling. Python handles it automatically.

⚡
Methods are just functions

The only difference: they live inside a class and take self as the first parameter.
