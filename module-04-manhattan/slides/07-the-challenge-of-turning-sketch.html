The Algorithm Knows Where, Not How

We have a path:

[(0,0), (1,0), (2,0), (2,1), (2,2)]

The robot knows WHERE to go. But HOW does it get there?

Going from (0,0) to (1,0) means "move one row down." But the robot needs to know:

Which direction am I facing RIGHT NOW?

Which direction do I NEED to face?

Do I need to turn? Which way?

Today we solve the turning puzzle.

What Is Heading?

Heading = the direction the robot is currently facing, stored as a number 0–3 going clockwise

Headings as Numbers (Clockwise)

Number Letter Meaning Row Δ Col Δ
0 N Facing up -1 0
1 E Facing right 0 +1
2 S Facing down +1 0
3 W Facing left 0 -1
HEADING_NAMES = ["N", "E", "S", "W"]

Use HEADING_NAMES[heading] to display the letter

Compass Diagram

Key idea: The robot always starts facing a known heading (e.g., 0 for North). Numbers go clockwise — one right turn adds 1.

Direction from Coordinates

Given two consecutive coordinates, what direction is the robot moving?

current  = (0, 0)
next_pos = (1, 0)

cur_row, cur_col = current
nxt_row, nxt_col = next_pos
row_diff = nxt_row - cur_row   # 1
col_diff = nxt_col - cur_col   # 0
row_diff col_diff Heading
-1 0 0 — North (up)
0 +1 1 — East (right)
+1 0 2 — South (down)
0 -1 3 — West (left)

Example: (2,0) → (2,1) — row_diff=0, col_diff=+1 → heading 1 (East)

Understanding the % (Modulo) Operator

What does % do?

The % operator gives the remainder after division.

7 % 4 = 3    # because 7 ÷ 4 = 1 remainder 3

The clock analogy

If it's 11 o'clock and you add 3 hours, it's 2 o'clock, not 14. That wrapping is what % 12 does. We do the same with % 4 for our four headings.

Negative numbers? No problem.

Python's % always gives a non-negative result when the divisor is positive.

(-1) % 4 = 3
(-2) % 4 = 2
(-3) % 4 = 1

How? Python finds the largest multiple of 4 that is ≤ the number, then subtracts:

(-1) % 4: largest multiple of 4 ≤ -1 is -4, so -1 - (-4) = 3

This is exactly what we want!

From → To Formula Right Turns
N(0) → S(2) (2 - 0) % 4 = 2 2 ✓
S(2) → N(0) (0 - 2) % 4 = 2 2 ✓
N(0) → E(1) (1 - 0) % 4 = 1 1 ✓
W(3) → N(0) (0 - 3) % 4 = 1 1 ✓
E(1) → S(2) (2 - 1) % 4 = 1 1 ✓
S(2) → E(1) (1 - 2) % 4 = 3 3 ✓
The % operator wraps negative results around to give us the correct clockwise count every time.

Turn Logic in Python

With numbers, the code is incredibly simple:

def count_right_turns(current, needed):
    return (needed - current) % 4

Then the robot executes:

turns = count_right_turns(heading, needed)
for i in range(turns):
    turn_right()      # sensor-based
heading = needed
Two lines of code handle every possible turn!

Updating Heading After Turning

After turning, what is the new heading? It is simply the needed heading!

The whole process:

needed = get_needed_heading(current_pos, next_pos)
turns = count_right_turns(heading, needed)
for i in range(turns):
    turn_right()
heading = needed

Why so simple? We always turn right until we face the needed direction. So the new heading IS the needed heading.

heading = needed   # That's it!
If you forget to update heading, subsequent turns will be wrong!
