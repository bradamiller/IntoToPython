Converting the Function

Side by side: function vs. method

Lesson 4 Function

def compute_path(start, dest):
    path = []
    cur_row, cur_col = start
    dest_row, dest_col = dest

    while cur_row < dest_row:
        cur_row = cur_row + 1
        path.append((cur_row, cur_col))
    # ... 3 more while loops

    return path

Lesson 5 Method

def compute_path(self, dest):
    path = []
    cur_row, cur_col = self.position
    dest_row, dest_col = dest

    while cur_row < dest_row:
        cur_row = cur_row + 1
        path.append((cur_row, cur_col))
    # ... 3 more while loops

    return path

What changed?

start parameter is gone — replaced by self.position

self is added as the first parameter

The 4 while loops stay exactly the same

Two changes: add self, use self.position instead of start. That's it!

The Complete Manhattan Class

class Manhattan:
    def __init__(self, start):
        self.position = start

    def compute_path(self, destination):
        path = []
        current_row, current_col = self.position
        dest_row, dest_col = destination

        while current_row < dest_row:
            current_row = current_row + 1
            path.append((current_row, current_col))
        while current_row > dest_row:
            current_row = current_row - 1
            path.append((current_row, current_col))
        while current_col < dest_col:
            current_col = current_col + 1
            path.append((current_row, current_col))
        while current_col > dest_col:
            current_col = current_col - 1
            path.append((current_row, current_col))

        return path

Key: Only one of each pair of while loops runs — if current_row < dest_row, then current_row is never > dest_row.

Testing the Class

Create an instance, call compute_path, print the result:

manhattan = Manhattan((0, 0))
path = manhattan.compute_path((2, 3))
print(path)
print("Steps:", len(path))

Output:

[(1, 0), (2, 0), (2, 1), (2, 2), (2, 3)]

Multiple Paths from Same Object

compute_path does NOT change self.position:

manhattan = Manhattan((0, 0))

path1 = manhattan.compute_path((2, 3))
print("Path 1:", path1)

path2 = manhattan.compute_path((1, 1))
print("Path 2:", path2)

print("Position:", manhattan.position)

Output:

Path 1: [(1, 0), (2, 0), (2, 1), (2, 2), (2, 3)]
Path 2: [(1, 0), (1, 1)]
Position: (0, 0)

Why? The method uses local variables (current_row, current_col) — not self.position.

Both paths start from (0, 0) because self.position never changes.

compute_path reads self.position but never modifies it — you can call it as many times as you want.

Edge Cases

What happens in special situations?

Same row — only columns change

manhattan = Manhattan((0, 0))
path = manhattan.compute_path((0, 3))
print(path)
# [(0, 1), (0, 2), (0, 3)]
print("Steps:", len(path))  # 3

The two row while loops do nothing (current_row already equals dest_row).

Same position — empty list

manhattan = Manhattan((2, 2))
path = manhattan.compute_path((2, 2))
print(path)
# []
print("Steps:", len(path))  # 0

All four while loops are skipped — no steps needed!

Reverse direction works too:

manhattan = Manhattan((3, 3))
path = manhattan.compute_path((1, 0))
print(path)   # [(2, 3), (1, 3), (1, 2), (1, 1), (1, 0)]
print("Steps:", len(path))  # 5
The 4 while loops handle every direction automatically — no special cases needed.
