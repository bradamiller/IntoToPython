How Do You Walk in a City?

"If you're at 1st Street & 1st Avenue and need to get to 3rd Street & 3rd Avenue, how do you walk?"

You can't cut diagonally through buildings!

Walk along streets (rows) and avenues (columns)

Go north/south first, then east/west -- or vice versa

This is Manhattan distance -- named after the grid layout of Manhattan, New York City. Our robot has the same constraint: it drives along grid lines, not diagonally.
2 / 11

Manhattan Distance

The total number of row steps + column steps between two points

Example 1

Start: (0, 0)

End: (2, 3)

Row distance: |2 - 0| = 2

Column distance: |3 - 0| = 3

Manhattan distance: 2 + 3 = 5 steps

Example 2

Start: (3, 3), End: (1, 0)

|1 - 3| + |0 - 3| = 2 + 3 = 5 steps

The Algorithm: Rows First, Then Columns

Strategy: Move along rows until correct row, then move along columns

Example: (0, 0) to (2, 3)

Phase 1: Move rows (0 → 2)

(0,0) → (1,0) → (2,0)

Phase 2: Move columns (0 → 3)

(2,0) → (2,1) → (2,2) → (2,3)

Path does NOT include start.

path = [(1,0), (2,0), (2,1), (2,2), (2,3)]

Steps = len(path) = 5

Part 1: The compute_path Function

Positive directions only -- south (row+1) and east (col+1)

def compute_path(start, end): path = [] current_row, current_col = start dest_row, dest_col = end while current_row < dest_row: current_row = current_row + 1 path.append((current_row, current_col)) while current_col < dest_col: current_col = current_col + 1 path.append((current_row, current_col)) return path

Key ideas

path = [] -- start with empty list

Tuple unpacking pulls out row and col

First while loop: move south (row increases)

Second while loop: move east (col increases)

Start position is NOT added to path

Calling it:

path = compute_path((0, 0), (2, 3)) print(path)

Tracing Part 1: (0,0) to (2,3)

Step-by-step trace

Setup:

path = [] current_row, current_col = 0, 0 dest_row, dest_col = 2, 3

While loop 1: current_row < dest_row

row 0 < 2: row=1, append (1,0)

row 1 < 2: row=2, append (2,0)

row 2 < 2: FALSE -- stop

While loop 2: current_col < dest_col

col 0 < 3: col=1, append (2,1)

col 1 < 3: col=2, append (2,2)

col 2 < 3: col=3, append (2,3)

col 3 < 3: FALSE -- stop

[(1,0), (2,0), (2,1), (2,2), (2,3)]

Part 1 Works... Sometimes

What happens with (3, 3) to (1, 0)?

Let's trace it:

current_row, current_col = 3, 3 dest_row, dest_col = 1, 0

While loop 1: current_row < dest_row

3 < 1? FALSE -- loop never runs!

While loop 2: current_col < dest_col

3 < 0? FALSE -- loop never runs!

Returns [] -- an empty list! The robot doesn't move at all.

The problem: our while loops only handle < (increasing). We need to also handle > (decreasing)!

Part 2: Adding North and West

Add 2 more while loops -- 4 total, no if/else needed!

def compute_path(start, end): path = [] current_row, current_col = start dest_row, dest_col = end while current_row < dest_row: # south current_row = current_row + 1 path.append((current_row, current_col)) while current_row > dest_row: # north current_row = current_row - 1 path.append((current_row, current_col)) while current_col < dest_col: # east current_col = current_col + 1 path.append((current_row, current_col)) while current_col > dest_col: # west current_col = current_col - 1 path.append((current_row, current_col)) return path

Why does this work?

Only ONE row loop runs (the other's condition is false)

Only ONE col loop runs (same reason)

If already on the right row, BOTH row loops skip

No if/else needed -- the while conditions do the choosing!

4 while loops, but at most 2 ever execute for any given call.

Test it:

path = compute_path((3, 3), (1, 0)) print(path)

Tracing Part 2: (3,3) to (1,0)

Step-by-step trace

current_row, current_col = 3, 3 dest_row, dest_col = 1, 0

Loop 1 (south): 3 < 1? FALSE -- skip

Loop 2 (north): current_row > dest_row

3 > 1: row=2, append (2,3)

2 > 1: row=1, append (1,3)

1 > 1: FALSE -- stop

Loop 3 (east): 3 < 0? FALSE -- skip

Loop 4 (west): current_col > dest_col

3 > 0: col=2, append (1,2)

2 > 0: col=1, append (1,1)

1 > 0: col=0, append (1,0)

0 > 0: FALSE -- stop

[(2,3), (1,3), (1,2), (1,1), (1,0)]

Edge Cases

Special situations the algorithm handles automatically

Same row -- only column loops run:

(2, 0) to (2, 3)

Both row loops skip (2 == 2)

path = [(2,1), (2,2), (2,3)]

Same column -- only row loops run:

(0, 1) to (3, 1)

Both col loops skip (1 == 1)

path = [(1,1), (2,1), (3,1)]

Same position -- all 4 loops skip:

(1, 1) to (1, 1)

path = [] (empty list)

len(path) = 0 -- already there!

No special cases needed. The while loop conditions handle everything.
