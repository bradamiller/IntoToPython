How Do You Walk in a City?

"If you're at 1st Street & 1st Avenue and need to get to 3rd Street & 3rd Avenue, how do you walk?"

You can't cut diagonally through buildings!

Walk along streets (rows) and avenues (columns)

Go north/south first, then east/west -- or vice versa

This is Manhattan distance -- named after the grid layout of Manhattan, New York City. Our robot has the same constraint: it drives along grid lines, not diagonally.
2 / 11

Manhattan Distance

The total number of row steps + column steps between two points

Example

Start: (0, 0)

Destination: (2, 3)

Row distance: |2 - 0| = 2 steps

Column distance: |3 - 0| = 3 steps

Manhattan distance: 2 + 3 = 5 steps

Manhattan distance is always ≥ straight-line distance. They're equal only when moving purely along one axis.

The Algorithm: Rows First, Then Columns

Strategy: Move along rows until correct row, then move along columns

Example: (0, 0) to (2, 3)

Step 1: Move along rows (0 → 2)

(0, 0) → (1, 0) → (2, 0)

Step 2: Move along columns (0 → 3)

(2, 0) → (2, 1) → (2, 2) → (2, 3)

(0,0) → (1,0) → (2,0) → (2,1) → (2,2) → (2,3)

Total: 5 steps (matches Manhattan distance!)

Tracing on the Grid

Visual: (0, 0) to (2, 3)

The path always makes an "L" shape: down 2 rows, then right 3 columns. Always rows first, then columns.
5 / 11

Direction Matters

What if the destination is UP or LEFT?

Example: (2, 3) to (0, 1)

Row: go from 2 to 0 (decrease -- move UP)

Column: go from 3 to 1 (decrease -- move LEFT)

(2,3) → (1,3) → (0,3) → (0,2) → (0,1)

How to determine direction:

dest row > current row → move DOWN

dest row < current row → move UP

dest col > current col → move RIGHT

dest col < current col → move LEFT

6 / 11

The Step Value

Use +1 or -1 to control direction

If dest row > start row:
    row_step = +1   # down
Else:
    row_step = -1   # up

If dest col > start col:
    col_step = +1   # right
Else:
    col_step = -1   # left

Example: (3, 0) to (1, 2)

Row: 3 → 1, so row_step = -1

Col: 0 → 2, so col_step = +1

Path: (3,0) → (2,0) → (1,0) → (1,1) → (1,2)

More Practice -- Hand Trace

Trace these paths by hand

Path 1: (0, 0) to (3, 2)

Row steps: 0 → 3 (step = +1): (0,0), (1,0), (2,0), (3,0)

Col steps: 0 → 2 (step = +1): (3,1), (3,2)

Full: (0,0) → (1,0) → (2,0) → (3,0) → (3,1) → (3,2)

Path 2: (2, 2) to (0, 0)

Row steps: 2 → 0 (step = -1): (2,2), (1,2), (0,2)

Col steps: 2 → 0 (step = -1): (0,1), (0,0)

Full: (2,2) → (1,2) → (0,2) → (0,1) → (0,0)

Path 3: (1, 0) to (1, 3)

Row steps: 1 → 1 (none needed -- already on correct row!)

Col steps: 0 → 3 (step = +1): (1,1), (1,2), (1,3)

Full: (1,0) → (1,1) → (1,2) → (1,3)

8 / 11

Edge Cases

Special situations the algorithm handles

Same row -- only column movement:

(2, 0) to (2, 3): just move right

Path: (2,0) → (2,1) → (2,2) → (2,3)

Same column -- only row movement:

(0, 1) to (3, 1): just move down

Path: (0,1) → (1,1) → (2,1) → (3,1)

Same position -- no movement:

(1, 1) to (1, 1): path is just [(1, 1)]

Manhattan distance = 0

All of these work with our algorithm! No special cases needed.
