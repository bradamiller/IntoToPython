Testing Strategy

manhattan = Manhattan((0, 0)) for dest in [(2,3), (0,1), (3,3), (1,0)]: path = manhattan.compute_path(dest) print(dest, "->", path) manhattan.position = dest

Step 2: Test Navigator with one short path

Start with (0, 0) to (1, 0) -- one step

Then (0, 0) to (2, 0) -- straight line

Then (0, 0) to (1, 1) -- requires one turn

Step 3: Test full sequence

Run all 4 destinations

Watch for accumulated heading errors

Common Issues & Fixes

Issue 1: Robot turns wrong direction

Does physical starting heading match the code?

Is turn_to() updating self.heading?

Print heading before and after each turn

Issue 2: Robot overshoots intersections

Is straight() distance correct for your grid?

Measure actual distance between intersections

Issue 3: Position not updating between legs

Check: manhattan.position = navigator.position

Without this, ALL paths start from (0, 0)!

Issue 4: Path correct on screen, wrong on robot

This is usually a turning or distance issue

Test Manhattan output first, then debug Navigator separately

Key debugging strategy: Always separate algorithm issues from physical robot issues. Test Manhattan on screen first!
